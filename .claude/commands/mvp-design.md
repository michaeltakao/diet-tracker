Design an MVP for the given product idea.

Work through this structure:

## 1. Customer Definition
- Who specifically has this problem? (not "everyone")
- What do they currently do instead?
- How often does this problem occur?
- How painful is it (1-10)?

## 2. Core Value
- One sentence: what does this product do for the customer?
- Why is this 10x better than the current alternative?

## 3. MVP Scope
Must have (validates the core value):
- Feature 1
- Feature 2

Must NOT have (premature complexity):
- Feature X — reason
- Feature Y — reason

## 4. Success Metric
- One primary metric that proves the MVP works
- Target value and timeframe

## 5. Kill Criteria
- What result, by when, proves this idea should stop?

## 6. First Validation Step
- What is the cheapest way to test the biggest assumption?
- Before writing code, what can be validated with a doc / landing page / conversation?

## 7. Key Assumptions
List the top 3 assumptions ranked by:
- Risk (how bad if wrong?)
- Uncertainty (how likely to be wrong?)
Validate the highest risk × highest uncertainty assumption first.
